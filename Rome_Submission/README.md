## Overview

This repository contains the submission for the ROME (RTL-Oriented Multi-step Evaluation) Demo Assignment, which explores the use of large language models for automated, hierarchical Verilog RTL generation. The assignment is divided into two parts: a provided multiplexer demo (Part I) and an independently designed ripple-carry adder hierarchy (Part II), both generated using the ROME framework with GPT-4o as the backend model.

All RTL was generated automatically using the `hier_gen` and `verilog_loop` functions from the ROME framework. No changes were made to the core framework code. Testbench files were written manually and used to verify correctness via iverilog and vvp simulation.

## Part II вЂ” Ripple Carry Adder Hierarchy (Extension Design)

The extension implements a four-level ripple-carry adder hierarchy following Option A. The complete instantiation chain is:

```
half_adder в†’ full_adder в†’ adder4 в†’ adder8
```

This hierarchy was verified by inspecting the compiled vvp binary, which confirmed that `adder8` contains `adder4_0` and `adder4_1`, each containing four `full_adder` instances (`fa0`вЂ“`fa3`), each containing two `half_adder` instances (`ha1`, `ha2`).

| Module      | Description              | Instantiates      | Iterations |
|-------------|--------------------------|-------------------|------------|
| half_adder  | Half adder (leaf)        | Nothing (leaf)    | 1          |
| full_adder  | Full adder               | half_adder Г— 2    | 1          |
| adder4      | 4-bit ripple carry adder | full_adder Г— 4    | 1          |
| adder8      | 8-bit ripple carry adder | adder4 Г— 2        | 3          |

> **Note on adder8 (3 iterations):** A prompt-construction limitation in `hier_gen` causes the last module in the submodules list to receive no updated prompt, resulting in GPT initially regenerating `adder4` instead of `adder8`. The self-correcting feedback loop passed the exact iverilog compiler error back to GPT-4o, which correctly inferred the missing top-level module requirement and generated `adder8` on the third iteration.
